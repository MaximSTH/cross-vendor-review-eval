Based on my analysis, the change removes the secret **value** from the `--credential` argument, passing only the bare env-var name `NVIDIA_API_KEY`. Every other `--credential` usage in the codebase supplies a `KEY=VALUE` pair:

- `onboard.js`: `--credential "OPENAI_API_KEY=dummy"`, `"OPENAI_API_KEY=ollama"`
- `runner.py:213`: `f"OPENAI_API_KEY={credential}"` (value resolved from env, then embedded)

There is no evidence openshell resolves a bare credential name from the environment. The intent (avoid leaking the key in `ps`/process args) is legitimate, but as written the actual API key value is never passed to openshell, so the `nvidia-nim` provider is created/updated without a working credential — silently, since the calls are guarded by `|| true` / `ignoreError`.
